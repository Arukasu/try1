The VPN

What exactly is a VPN?

To fully understand what a VPN is, you must first understand how transferring data works.

For example, imagine a phone playing videos from YouTube.

  1. The first action that occurs is clicking on the video. Data from your phone is transferred over to the YouTube servers telling what video they are going to send over to you.
  2. The next action is the video data being transmitted fromthe YouTube servers to your phone.

Now, this system works effectively, but there is a small issue. The YouTube servers must know where to send this data so the phone can receive it. This is done with an IP Address which is a line of numbers that shows the location of a specific device on the internet, much like addresses do in real life. Tools like ipleak.net show that your IP Address is actually exposing your location to the sites that you visit.

The solution to this dilemma is the VPN. A VPN, or a Virtual Private Network is a way to use the internet without exposing your actual IP Address.

Essentially, all the data that was going directly to your phone is now going directly to the VPN server. Then the VPN directly transfers the data to you so that your actual IP Address is untraceable. The only thing that the websites have access to is your VPN server's IP. This means that your location appears to be where the VPN server is located. This is why VPNs are often used to access region restricted content. For example, if you were in Europe and your favorite TV show was blocked on Netflix, you could connect to a United States VPN server to view the show since it tricks Netflix into thinking your location is in the US.

VPN Protocols

What protocol should I be using?

OpenVPN TCP  

+ Great connection reliability
+ Very popular
- Lacks speed

OpenVPN UDP  

+ Great speed
+ Very popular
- Lacks connection stability

Wireguard

+ Much faster than OpenVPN
+ Great connection reliability
- Not as many features

SoftEther

+Much faster than OpenVPN
+Protocol can bypass most firewalls
- Harder to install
- Obscure

IKEv2

+ Possibly faster than OpenVPN
- Has potentially been exploited in the past
- Limited compatability without using specific packages

What is a VPN gateway?

A VPN gateway is essentially a normal VPN connection which allows multiple devices in a network to connect to it.  

This is especially useful for those who would like their entire home network to be connected to a VPN server.

How to Make a VPN Gateway

Tutorial adapted and updated from HERE

    Requirements: Raspberry Pi 3 or newer with an SD card for storage
  1. Install Raspbian on the SD card. Select the "Lite" version to install.https://www.raspberrypi.org/software/operating-systems/
  2. Connect Using SSH. Start up the Raspberry Pi and connect a monitor, mouse, and keyboard. ​Open the console and type sudo systemctl enable ssh
     then sudo systemctl start ssh.
     This allows you to control the Raspberry Pi through Putty which uses ssh to access the commandline from another computer. Also enter
    ifconfig
    and write down the IP Address of the Pi. Start up Putty and connect to the Pi using the IP Address. The default username and password should be "pi" and "raspberry". This should be changed later for security.
  3. Configure Static IP Address for the RPI. 
    Identify the router address by typing
    ip r | grep default
    The result should look like the following:
    "default via 192.168.0.1 dev eth0 proto dhcp src 192.168.0.159 metric 202"
    In this case, the router address is "192.168.0.1".
    Next, type in the following:
    sudo nano /etc/dhcpcd.conf"
    Edit the following text:
    interface <NETWORK>
    static ip_address=<STATICIP>/24
    static routers=<ROUTERIP>
    static domain_name_servers=<DNSIP>
    Replace “<NETWORK>” with ​“eth0” (Ethernet) or "wlan0" (WiFi) depending on your connection type.
    Replace “<STATICIP>” with the IP address of your Pi.
    Replace “<ROUTERIP>” with the router address found previously.
    Replace “<DNSIP>” with  Google's “8.8.8.8” or CloudFlare's “1.1.1.1“ DNS address.
    Save the file by pressing CTRL + X then Y followed by ENTER.
  4. Install Upgrades
    Type in:
    sudo apt update && sudo apt upgrade -y && sudo apt dist-upgrade -y && sudo apt autoremove -y
    While installing iptables-persistent, it might ask if you want to backup the iptables rules. Select Yes.
  5. Setting Up NordVPN
    Type in this line by line:
    cd /etc/openvpn
    sudo su
    sudo wget https://downloads.nordcdn.com/configs/archives/servers/ovpn.zip
    sudo unzip ovpn.zip
    sudo cp ovpn_udp/*.ovpn .
    sudo rm -rf ovpn_tcp ovpn_udp
    This only uses the UDP connections.
    Next, create a file with your NordVPN account username and password
    sudo nano /etc/openvpn/login.txt
     Format the file like:
    username
    password
    Next, protect it by typing
    sudo chmod 600 /etc/openvpn/login.txt
    Create a Python file with the contents:
    #!/usr/bin/env python
    import fileinput
    import glob
    import os
     os.chdir("/etc/openvpn")
    file_list = glob.glob("*.ovpn")
    for item in file_list:
        for line in fileinput.input(item, inplace = 1):
            print line.replace("auth-user-pass", "auth-user-pass login.txt"),
    Run the python script so that the .ovpn files will automatically accept your login credentials
  6. Select a NordVPN location
    Visit the NordVPN server tools page where you can find the most optimal server.
    Open Developer Tools in your browser and click on XHR









    select the country that you want to use for the VPN connection and observe the XHR request that gets generated. Right click on the XHR request and copy the URL. The URL will be used in the next step.











    For example, Australia's would be
    https://nordvpn.com/wp-admin/admin-ajax.php?action=servers_recommendations&filters={%22country_id%22:13}
    7. Create NordVPN Connection Script
    Now we need to tell the RPi to connect to the NordVPN server on boot.
    Here is a python script outline that does this. Save it as /home/pi/vpn/connector.py.
    Make sure to fill the "URL" field with the country's URL you wish to use from NordVPN.
    #!/usr/bin/env python
    import os
    import json
    import subprocess
    import time
    import requests os.chdir( "/etc/openvpn" )

    # Command to kill any running instances of OpenVPN
    kill_command =     "sudo killall openvpn"

    # URL to the NordVPN server connection tool obtained from the browser
    url =     "" # Insert URL here
    def start_openvpn_connection () :
        response = requests.get(url)
        if len(response.text) != 2 :
            nvpn_response = json.loads(response.text)
            vpn_info = nvpn_response[     0 ]
            vpn_info_hostname = vpn_info[     "hostname" ]
            vpn_file = vpn_info_hostname +     ".udp.ovpn"

            # Command to start Openvpn
            ov_command =     "sudo openvpn --config " + vpn_file

             # Start the NordVPN connection
             subprocess.Popen(ov_command.split())
    if __name__ == "__main__" :
        subprocess.Popen(kill_command.split())
        time.sleep(     2     )
        start_openvpn_connection()
    Next you need to configure so it will run at startup.
    Open /etc/rc.local with
    sudo nano /etc/rc.local
    Insert the following script before the "exit 0" line:
    python /home/pi/vpn/connector.py
  7. Find NordVPN Server IPs
    Enter:
    sudo apt-get install dnsutils
    Wait for the package to install.
    Type in
    nslookup nordvpn.com
    Then write down the two addresses under the non-authoitative answer.
  8. Add NordVPN IP Addresses to Hosts File and Block Outbound DNS
    Open /etc/hosts:
    sudo nano /etc/hosts
    Add the below lines and replace <nordaddress1> and <nordaddress2> with the two addresses written down previously:
    <nordaddress1> nordvpn.com

    <nordaddress2> nordvpn.com
  9. Set Up IP Forwarding
    Enter in:
    sudo nano /etc/sysctl.conf
    Uncomment the line that says "net.ipv4.ip_forward=1"
    Save and exit then run the command:
    sysctl -p /etc/sysctl.conf
  10. Set IPTables Rules
    Remember to replace <lansubnet> with the subnet mask of the RPi. This can be found with the ifconfig command.
    Also replace the <nordaddress1> and <nordaddress2> with the Nord IP Addresses found in step 8.
    Run each line one by one in command line.
    sudo iptables -F
    sudo iptables -A INPUT -p tcp --dport 22 -m conntrack --ctstate NEW,ESTABLISHED -j ACCEPT 
    sudo iptables -A OUTPUT -p tcp --sport 22 -m conntrack --ctstate ESTABLISHED -j ACCEPT 
    sudo iptables -t nat -A POSTROUTING -o tun0 -j MASQUERADE 
    sudo iptables -A FORWARD -i tun0 -o eth0 -m state --state RELATED,ESTABLISHED -j ACCEPT 
    sudo iptables -A FORWARD -i eth0 -o tun0 -j ACCEPT 
    sudo iptables -A OUTPUT -o tun0 -m comment --comment "vpn" -j ACCEPT 
    sudo iptables -A OUTPUT -o eth0 -p icmp -m comment --comment "icmp" -j ACCEPT 
    sudo iptables -A OUTPUT -d <lansubnet> -o eth0 -m comment --comment "lan" -j ACCEPT 
    sudo iptables -A OUTPUT -o eth0 -p udp -m udp --dport 1194 -m comment --comment "allow vpn traffic" -j ACCEPT 
    sudo iptables -A OUTPUT -o eth0 -p udp -m udp --dport 123 -m comment --comment "ntp" -j ACCEPT 
    sudo iptables -A OUTPUT -p tcp -d <nordaddress1> --dport 443 -m comment --comment "nordvpn IP" -j ACCEPT 
    sudo iptables -A OUTPUT -p tcp -d <nordaddress2> --dport 443 -m comment --comment "nordvpn IP" -j ACCEPT 
    sudo iptables -A OUTPUT -o eth0 -j DROP 
    sudo iptables -I FORWARD -i eth0 ! -o tun0 -j DROP 
    sudo iptables-save | sudo tee /etc/iptables/rules.v4
  11. Reboot the RPi
    The gateway is now completed. In order to connect to it, you must set your default gateway to the IP Address of the RPi.
